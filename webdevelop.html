Planning

1.1 Inspirations

The main thing to consider when creating the website is how to get the userâ€™s attention. I did significant research and above is a screenshot of the designs I like the best. The elegant design, layout and content all grabbed my attention. Also, the left side navigation is very handy as it is faster to navigate.
Thus, I decided my website will have the left side bar menu, and I'm also thinking about a whole website pinkish colour palette with black elements or just a dark mode website with border aesthetic blue colour.

1.2 Wireframe Sketches

This is my single page desktop and mobile wireframe sketches. I sketched two variations. The right- hand side of my first variation desktop view sketch is a home page with small clickable images to navigate to that project page. I also made a portfolio page wireframe. I have a plan to make the animation with some of my created models. Thus, it should be in this page on the whole screen, and it will be clickable to go to the projects page.
Afterwards, I saw no reason to create an extra page when all the projects are already on the home page with a link to that project. I therefore did some more research and found the horizontal scroll page websites.
So, I made another variation sketch, which is on the left side hand. There is the home page with bigger images, and the other portfolio page will slide horizontally when the user wheels the mouse. In that page there will be images containing a tag to that project. I think it is a good idea, as it saves a lot of vertical screen space.

1.3 Digital Wireframes

I have made a couple more wireframes by using Adobe XD software. However, I chose the second, a hand sketch version with a horizontal scrolling portfolio page. This is better than the previous and last versions of the arrangement of the website content. I'm happy about the horizontal scrolling page. The users will see my creation images when they just visit a page, and on the right-hand side the latest project. As I mentioned, it is a clickable image. On the click, the user will enter to explore that project, and when a guest wheels a mouse, it will see the multiple projects.
My priority was to avoid adding too much content. I did not want to make a busy website. However, on the previous wireframes I made the page too busy. It was too much content, and the small images made it look overcrowded on the page.
Thus, on this wireframe I only included the logo image of where I study and text about me, my hard skills.

1.4 Mobile App Wireframe

The navigation I placed on the top. I found the top nav bar has better visibility for mobile devices, and I agree with that. When I experimented with different website navigation bars (side, bottom, top), the top links grabbed my attention.
Mobile users will not see the projects page, as they see in the desktop version. The screen is much smaller, but by saving the home page screen space, the horizontal portfolio page, the guests will only see when they scroll by touching the screen.
Also, I'm planning to make another animation for the contact page. I did a video wireframe on the desktop version. But to avoid overloading the mobile device, I did not add it. Thus, on small devices, there will be just a contact form and the centred image.

1.5 Mockup

For this step I chose to see my idea of having grey line wireframes in visual, to make more realistic wireframes with details, such as colours, buttons, typography, to replace the grey layouts to real content.
But before that, I created a website without a mockup and I was unhappy with the results, and stuck on the design, sizing, navigation.
Thus, I decided to recreate the website, and the mockup helped. This new design was developed rapidly. The result is much better. I changed the base pinkish colours to the dark grey gradient. By reducing some box elements in the backgrounds and adding some borders, my website looks more elegant, which is what I wanted.

Prototype

To avoid my previous mistakes. I just followed all these steps. The process of designing the site before starting to build and launch the website saved me time.
This step helped me refine the website design. I observed the pinkish social media link to my CV did not work with my website, so I changed it to grey. Also, I reduced the amount of the boxes from the contact page as it looked too overcrowded, and as well I changed the project web development cover image.

Execution

My previous versions of the site had many problems. Margins, positions, images are displayed only on the local device, and it was not responsive. I was using all content in pixels. This happened because I didn't do my research properly.
My new website is the result of my experiential learning. I could have built it earlier if I hadn't used code from previous sites. which were built in a complex way, and if I had used Bootstrap.
On this website, the horizontal project page is not working, so I added a scroll bar. I also haven't found a solution to fix the home page images flickering at the beginning.
